Inicios GIT

Git es un sistema de control de versiones distribuido, gratuito y de código abierto, diseñado para manejar proyectos pequeños y grandes con velocidad y eficiencia.

Instrucciones de GIT

  1. Se abre GIT desde la raiz del proyecto
  2. Se ejecuta el comando git init, para iniciar el GIT y poder trabajar
  3. Se ejecuta el comando git add, para agregar los archivos y cambios al repositorio .
  4. Se ejecuta el comando git commit -m "Mensaje del commit", para dejar el comentario de lo que se edito
  5. Se ejecuta el comando git remote add origin URL
  6. Se ejecuta el comando git push -u origin main

Panoramas

  1. Varios cambios en diferentes archivos creados: se pueden escribir los nombres de los arvhos quue modifico, por ejemplo: "git add index.html contacto.html" enter y luego se hace el commit para dejar los comentarios.
  2. Si se quiere agregar todos los cambios de una vez, se puede escribir "git add ." y luego se hace el commit para dejar los comentarios.
  3. Si se quiere crear un commit para cambios por separado, se tiene que agreagar al stage el archvo que se quiere guardar, es decir, si se modifico mas de un archivo pero solo se quiere guardar los cambos en uno solo, se puede hacer con git add y el nombre del archivo.

Ramas en GIT

  1. Se ejecuta git branch para ver las ramas que existen
  2. Se ejecuta git branch nombreRama para crear una nueva rama
  3. Se ejecuta git checkout nombreRama para cambiar de rama
  4. Se ejecuta git merge nombreRama para unir la rama actual con la rama que se indique
  5. Se ejecuta git branch -d nombreRama para eliminar una rama

conflictos en GIT

  1. Se ejecuta git merge main para unir la rama actual con la rama main
  2. En caso de que se haya editado el proyecto en la misma linea, en 2 diferentes ramas, al hacer el merge hay un conficlto, VSC lo puede arreglar por si mismo
  3. Si VSC no puede arreglar el conflicto, se tiene que hacer manualmente, se tiene que abrir el archivo y buscar las lineas que tienen conflicto, se tiene que borrar las lineas que no se quieren y dejar las que si, se tiene que borrar las lineas que tienen <<<<<<<,=======y>>>>>>>, y dejar solo las lineas que se quieren.
  4. Se ejecuta git add . para agregar los cambios y luego se ejecuta git commit -m "Mensaje del commit" para dejar el comentario de lo que se edito
  5. Se ejecuta git push origin nombreRama para subir los cambios a la rama que se indique

Devolverse a los commits

  1. Se ejecuta git log para ver los commits que se han hecho
  2. Se ejecuta git checkout y se agrega el "hash", son las letras amarrilas sobre arriba de cada commit, identificador publico
  3. Se ejecuta git checkout main para volver a la rama principal

Reset

  1. Se ejecuta git log para ver los commits que se han hecho
  2. Se selecciona el commit al que se quiere volver usanado el "hash", son las letras amarrilas sobre
  3. Se usa "git reset [id del commit]", o bien, hay 3 id: "--mixed" "--soft" "--hard" cualquiera de ellos se puede usar, pero se usa antes del [id del commit (hash)]
  4. Se ejecuta git push origin main para subir los cambios a la rama principal

    Tipos de reset

  • git reset --mixed [id del commit]: Se descartan los cambios, sin embargo no estan en el stage, tipo como si no se hubiese hecho ningun commit, despues de ejecutar el comando, se puede usar "git restore ." para descartar los cambios.
  • git reset --soft [id del commit]: los cambios se encuentran dentro del stage, para borrar los cambios se tiene que usar "git restore --staged" para removerlo del stage
  • git reset --hard [id del commit]: se considera un "comando destructivo", una vez se ejecute elimina los cambios hasta el commit que usted decidio, es util cuando se esta seguro que los cambios se van a perder. Sin embargo, se pueden recuperar los cambios usando git reflog

GIT reflog

  1. Se ejecuta git reflog para ver un historial, permite visualizar los movimientos que se hicieron en el repositorio
  2. Se busca el commit (hash)
  3. Se copia el hash y se usa: "git reset --hard [hash]" y enter

GIT TAG

Los tags son una forma de marcar un punto en la historia de un repositorio, se pueden usar para marcar versiones, lanzamientos, etc.

  1. Se ejecuta git tag para ver los tags que existen
  2. Se ejecuta git tag v1.0 -m "Mensaje del tag" para crear un tag, se pueden usar de otras formas (android usa nombres de dulces, etc). En caso de no hacer cambios grandes, se puede usar el v1.0.1. En caso de agreagar fucniones mas avanzadas v1.1.1, en caso de un cambio mas grande se puede usar v2.1.1
  3. Se ejecuta git push origin v1.0 para subir el tag a la rama principal
  4. En caaso de querer elimiar un tag, se usa: git tag -d [nombre del tag]
  • Se pueden agregar tags a determinados commits
  • Se ejecuta git tag -a v1.0 [hash] -m "Mensaje del tag" para agregar un tag a un commit en especifico
  • Se puede ver infomacion del tag con git show v1.0

Repositorio

Un repositorio es un espacio donde se almacenan los archivos de un proyecto, se puede tener un repositorio local y uno remoto.

  1. Se ejecuta git clone URL para clonar un repositorio
  2. Se ejecuta git pull para traer los cambios del repositorio remoto al repositorio local
  3. Se ejecuta git push para subir los cambios del repositorio local al repositorio remoto

GIT FETCH

Git fetch es un comando que se usa para traer los cambios del repositorio remoto al repositorio local, sin embargo, no se mezclan los cambios, es decir, no se hace un merge.

  1. Se ejecuta git fetch origin para traer los cambios del repositorio remoto al repositorio local, el fectch hace una comparacion de nuestros archivos locales con los del servidor, en caso de existir diferencia, nos pedira realizar un git pull
  2. Se ejecuta git merge origin/main para mezclar los cambios del repositorio remoto al repositorio local

git clone

Git clone es un comando que se usa para clonar un repositorio remoto a un repositorio local.

  1. Se ejecuta git clone URL para clonar un repositorio remoto a un repositorio local
  2. Se ejecuta git clone URL nombreCarpeta para clonar un repositorio remoto a un repositorio local con un nombre de carpeta diferente
